
1. Define a pointcut on your service method

@Aspect
@Component
public class PropertyValidationAspect {

    private final Validator validator;

    public PropertyValidationAspect(Validator validator) {
        this.validator = validator;
    }

    // Pointcut matching your service method
    @Pointcut("execution(* simple.chatgpt.service.mybatis.PropertyServiceImpl.updateProperty(..))")
    public void updatePropertyPointcut() {}

    // Advice before the method runs
    @Before("updatePropertyPointcut() && args(key, newValue)")
    public void validateProperty(PropertyKey key, String newValue) {
        Property prop = new Property(key.getKey(), key.getTypeName(), newValue);

        Set<ConstraintViolation<Property>> violations = validator.validate(prop);

        if (!violations.isEmpty()) {
            StringBuilder sb = new StringBuilder();
            for (ConstraintViolation<Property> v : violations) {
                sb.append(v.getMessage()).append("; ");
            }
            throw new IllegalArgumentException("Validation failed: " + sb.toString());
        }
    }
}

///////////////////

In Spring AOP, you have two styles of declaring advice:

Annotation-based (using @Aspect, @Before, @Around, etc.)
→ Needs only <aop:aspectj-autoproxy/> in XML.
→ Cleaner, less verbose, what I showed earlier.

XML-based AOP configuration (no @Aspect annotation).
→ Everything is defined in applicationContext.xml.
→ More old-school, but still works fine in non-Boot setups.

////////////////

If you want XML-style advice for your case (updateProperty(PropertyKey, String)):

1. Add your validator aspect bean

Put this near your other <bean> definitions:
<bean id="propertyValidationAspect" class="simple.chatgpt.aop.PropertyValidationAspect"/>

2. Add an AOP config block for it

<aop:config>
    <!-- Pointcut for updateProperty in your service -->
    <aop:pointcut id="propertyUpdatePointcut"
                  expression="execution(* simple.chatgpt.service.mybatis.PropertyServiceImpl.updateProperty(..))"/>
    
    <!-- Advice runs before method execution -->
    <aop:aspect ref="propertyValidationAspect">
        <aop:before method="validateProperty" pointcut-ref="propertyUpdatePointcut"/>
    </aop:aspect>
</aop:config>


3. Your aspect class (no @Aspect needed, plain bean)

package simple.chatgpt.aop;

import org.aspectj.lang.JoinPoint;
import simple.chatgpt.pojo.mybatis.Property;
import simple.chatgpt.pojo.mybatis.PropertyKey;

import javax.validation.*;
import java.util.Set;

public class PropertyValidationAspect {

    private final Validator validator;

    public PropertyValidationAspect() {
        ValidatorFactory factory = Validation.buildDefaultValidatorFactory();
        this.validator = factory.getValidator();
    }

    // Called before updateProperty
    public void validateProperty(JoinPoint joinPoint) {
        Object[] args = joinPoint.getArgs();
        PropertyKey key = (PropertyKey) args[0];
        String newValue = (String) args[1];

        Property prop = new Property(key.getKey(), key.getTypeName(), newValue);

        Set<ConstraintViolation<Property>> violations = validator.validate(prop);
        if (!violations.isEmpty()) {
            throw new IllegalArgumentException("Validation failed: " + violations);
        }
    }
}
