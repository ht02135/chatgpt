
You currently have three primary domain entities (POJOs):

POJO	Database Table	Purpose
RoleManagementPojo	role_management	Defines a single role
RoleGroupManagementPojo	role_group_management	Defines a logical group of roles
PageRoleGroupManagementPojo	page_role_group_management	Defines which role-group can access which URL pattern

Because your data model also includes link (mapping) tables for many-to-many relationships.

These tables are not represented as standalone “core POJOs”, but they still need mappers for database CRUD.

Here are your mapping tables:

Mapping	Purpose	Typical Table Name	XML Mapper
Role ↔ RoleGroup	Which roles belong to which role-group	role_group_role_mapping	RoleGroupRoleMappingMapper.xml
User ↔ RoleGroup	Which users belong to which role-group	user_management_role_group_mapping	UserManagementRoleGroupMappingMapper.xml

///////////

These exist to maintain associations, not as “main” entities with behavior — but MyBatis still needs explicit mapper definitions for them.
