🧠 Step 2️⃣ — What you want instead

You want pattern matching like Spring’s AntPathMatcher,
which supports:

/property/** matching /management/jsp/property/properties.jsp

/user/* matching /user/123

🧩 Step 3️⃣ — Fix getAllowedRoleGroups(url)

Here’s what you can do inside your PageRoleGroupServiceImpl:

@Override
public List<String> getAllowedRoleGroups(String url) {
    logger.debug("getAllowedRoleGroups called with url={}", url);

    AntPathMatcher matcher = new AntPathMatcher();

    // load all mappings (you can cache later)
    List<PageRoleGroupManagementPojo> all = pageRoleGroupManagementService.getAll();
    logger.debug("getAllowedRoleGroups totalMappings={}", all.size());

    List<String> matchingRoleGroups = new ArrayList<>();

    for (PageRoleGroupManagementPojo p : all) {
        String pattern = p.getUrlPattern();
        logger.debug("getAllowedRoleGroups checking pattern={}", pattern);

        // Match ignoring any "/management/jsp/" prefix differences
        if (matcher.match(pattern, url)) {
            logger.debug("getAllowedRoleGroups MATCHED pattern={} for url={}", pattern, url);
            matchingRoleGroups.add(p.getRoleGroupRef());
        }
    }

    logger.debug("getAllowedRoleGroups matchingRoleGroups={}", matchingRoleGroups);
    return matchingRoleGroups;
}
