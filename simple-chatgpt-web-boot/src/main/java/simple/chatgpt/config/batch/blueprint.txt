| Old XML Feature                       | New Java Config Equivalent                                      |
| ------------------------------------- | --------------------------------------------------------------- |
| `<job>` with steps & decision         | `@Configuration` class defining a `Job` bean with `StepBuilder` |
| `<CronTriggerBean>` + `<JobDetail>`   | `QuartzSchedulerConfig` with `JobDetail` and `Trigger`          |
| `<tasklet>`, `<chunk>`                | Modern `Tasklet` or `ChunkOrientedTasklet` beans                |
| `<property>`/`<ref>` wiring           | Spring Boot / `@Bean` dependency injection                      |
| MyBatis readers/writers               | Still usable — just injected via beans                          |
| XML imports (ftp.xml, fileNaming.xml) | Centralized config classes (`@Configuration`)                   |
| `PropertyPlaceholderConfigurer`       | Use `application.yml` / Spring `@Value` injection               |

| Old (XML)                  | New (Spring Batch 5.2.3)                                           |
| -------------------------- | ------------------------------------------------------------------ |
| `<job>` / `<step>` XML     | Java DSL via `JobBuilder`, `StepBuilder`                           |
| `CronTriggerBean` (Quartz) | `@Scheduled(cron = "...")`                                         |
| `jobDetail` beans          | Replaced with `JobLauncher` call                                   |
| XML listeners              | Implement `JobExecutionListener` / `StepExecutionListener` classes |
| `scope="step"` beans       | Annotate with `@StepScope` in Java config                          |

Overall Design — English View
You want to run a batch job that:
Tracks a batch process header
Loads user_management records
Populates related tables (user_management_list, user_management_list_member)
Generates .csv output
Encrypts and transfers files (FTP)
Updates statuses accordingly

Folder Structure

You can keep everything under your existing web project, in a dedicated batch package:

src/main/java/com/example/batch
├── config
│   ├── BatchConfig.java            # Spring Batch job/step config
│   ├── DataSourceConfig.java       # DB datasource config
│   └── QuartzSchedulerConfig.java  # Scheduler config
├── job
│   ├── JobLauncherQuartzAdapter.java  # Quartz adapter to launch jobs
│   ├── userListJob
│   │   ├── UserListJobConfig.java         # Job definition, wires all steps
│   │   ├── Step1CreateBatchHeader.java    # Tasklet + StepExecutionListener
│   │   ├── Step2LoadUsers.java            # Tasklet + StepExecutionListener
│   │   ├── Step3PopulateUserList.java     # Tasklet + StepExecutionListener
│   │   ├── Step4GenerateCsv.java          # Tasklet + StepExecutionListener
│   │   ├── Step5EncryptAndTransfer.java   # Tasklet + StepExecutionListener
└── service
    └── BatchService.java                  # Shared services for steps


| Step  | Name                        | Type     | Implementation Class             | Description                                                                                            |
| ----- | --------------------------- | -------- | -------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **1** | Create Batch Process Header | Tasklet  | `Step1CreateBatchHeaderTasklet`  | Inserts a record into `batch_process` (custom table you define) and puts header ID in ExecutionContext |
| **2** | Load Users                  | Tasklet  | `Step2LoadUsersTasklet`          | Loads active users from `user_management`, filters/validates them                                      |
| **3** | Populate User List          | Tasklet  | `Step3PopulateUserListTasklet`   | Inserts records into `user_management_list` and `user_management_list_member`                          |
| **4** | Generate CSV                | Tasklet  | `Step4GenerateCsvTasklet`        | Creates `.csv` file in `/output/` directory                                                            |
| **5** | Encrypt and Transfer        | Tasklet  | `Step5EncryptAndTransferTasklet` | Uses PGP encryption and transfers to FTP server                                                        |
| **6** | Job Listener                | Listener | `UserListJobListener`            | Logs start/end, updates `batch_process` status                                                         |

| Category   | Class                           | Description                               |
| ---------- | ------------------------------- | ----------------------------------------- |
| Config     | `BatchConfig.java`              | Enables Batch                             |
| Config     | `DataSourceConfig.java`         | Defines 2 datasources (main + batch meta) |
| Job Config | `UserListJobConfig.java`        | Wires steps together                      |
| Steps      | `Tasklet` classes               | Each step logic                           |
| Listener   | `UserListJobListener.java`      | For start/end logs                        |
| Launcher   | `JobLauncherQuartzAdapter.java` | For scheduling or manual launch           |
| Service    | `BatchService.java`             | To trigger jobs programmatically          |


///////////////////

| File                                     | Purpose                                                                                                                         | How it’s connected                                                                                      |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| **`BatchConfig.java`**                   | Enables Spring Batch (`@EnableBatchProcessing`) and registers all batch infrastructure beans (JobRepository, JobLauncher, etc.) | Makes Spring aware of batch context; allows `JobBuilderFactory` and `StepBuilderFactory` to be injected |
| **`UserListJobConfig.java`**             | Declares the **Job** and all **Steps**                                                                                          | Spring automatically detects it (`@Configuration`) and registers the `userListJob` bean                 |
| **`Step1CreateBatchHeaderTasklet.java`** | Implements logic for Step 1                                                                                                     | Detected automatically (`@Component`) and injected into the correct Step via parameter matching         |
| **`UserListJobListener.java`**           | Listens to job lifecycle events                                                                                                 | Detected automatically (`@Component`) and injected into the Job as a listener                           |
| **`JobLauncherQuartzAdapter.java`**      | (To be added soon) — launches the job manually or on a schedule                                                                 | It gets a `JobLauncher` and the `userListJob` injected                                                  |
| **`BatchService.java`**                  | (Optional helper) to trigger jobs programmatically from web layer                                                               | Gets `JobLauncher` and `Job` injected and calls `run()`                                                 |

In Spring Batch, each Step can indeed have:

A Tasklet (or a reader/processor/writer if it’s a chunk step)
Optionally, Step Listeners

Tasklet / Task — the “worker bee”

Think of the tasklet as the piece of code that actually does the work.
| Step  | Tasklet Action                                                       |
| ----- | -------------------------------------------------------------------- |
| Step1 | Create a batch header in the DB and store its ID                     |
| Step2 | Load users from `user_management` table into memory or staging table |
| Step3 | Populate the user list table with those users                        |
| Step4 | Generate a CSV file from the user list                               |
| Step5 | Encrypt the CSV and send it to FTP                                   |

Listener — the “observer / watchdog”

A listener is not doing the main work. It’s watching and reacting to lifecycle events.

It can run before the step/job or after the step/job.
