1>Roles

roles.jsp → list all roles
addRole.jsp → create a new role
editRole.jsp → edit an existing role

2>Role Groups

roleGroups.jsp → list all role groups
addRoleGroup.jsp → create a new role group
editRoleGroup.jsp → edit an existing role group

3>Role-Group Mapping

This is associating roles with role groups. You can handle this in a few ways:

Either integrate with Role Groups UI (as a tab or modal inside roleGroups.jsp)
Or separate pages if you want a dedicated management interface:

roleGroupRoles.jsp → list roles in a role group
addRoleGroupRole.jsp → add role to a group
editRoleGroupRole.jsp → edit mapping (rarely needed)

4>age-Role Groups

pageRoleGroup.jsp → list page-role-group mappings
addPageRoleGroup.jsp → add new page-role-group
editPageRoleGroup.jsp → edit existing mapping

5>User-Role Groups

userRoleGroup.jsp → list user-role-group mappings
addUserRoleGroup.jsp → assign a user to a role group
editUserRoleGroup.jsp → edit assignment (optional, sometimes just delete & re-add)

Page ↔ RoleGroup Mapping

User ↔ RoleGroup Mapping
Operations:
Assign user to role group
Remove user from role group
List users per role group
Slightly more dynamic because users might be assigned frequently.
Cache invalidation exists, but UI logic is still standard.
✅ Verdict: Simple-to-moderate, still manageable in a single controller if needed.

For now, treat mappings as simple and fold them into the main controllers (RoleGroupManagementController, PageRoleGroupController, UserRoleGroupController).
